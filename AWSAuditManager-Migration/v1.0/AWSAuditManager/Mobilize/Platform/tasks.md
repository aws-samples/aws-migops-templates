
# Module: Platform
## Task 1: Understand the shared responsibility model
#### Description
Security is a shared responsibility between AWS and you. The shared responsibility model describes this as security of the cloud and security in the cloud:
* Security of the cloud – AWS is responsible for protecting the infrastructure that runs AWS services in the AWS Cloud. AWS also provides you with services that you can use securely.
* Security in the cloud – Your responsibility is determined by the AWS service that you use. You are also responsible for other factors including the sensitivity of your data, your company’s requirements, and applicable laws and regulations
#### Acceptance Criteria
Knowledge of the shared responsibility model in AWS for compliance migrations.
## Task 2: Create a data protection strategy
#### Description
To safeguard data in AWS, you must establish and implement a methodology for the encryption of your AWS resources. Use the following steps and create subtasks if needed.
* Document the current and readiness states of your data protection.
* Conduct a data protection readiness gap analysis and story development.
* Define an encryption-at-rest strategy.
* Define an encryption-in-transit strategy.
* Develop encryption security controls.
#### Acceptance Criteria
Define an AWS resource encryption strategy for data at rest.
Define an AWS resource encryption strategy for data in transit.
Complete a data protection gap analysis.Identify a data protection strategy.
## Task 3: Create governance and compliance strategy
#### Description
When you perform a migration to AWS, the responsibility for compliance is determined by the sensitivity of the data, your compliance objectives, and applicable laws and regulations. AWS provides the following resources to help with compliance:
* Security and Compliance Quick Reference guides - These deployment guides discuss architectural considerations, and provide steps for deploying security- and compliance-focused baseline environments on AWS.
* Architecting for HIPAA Security and Compliance whitepaper – This whitepaper describes how you can use AWS to create HIPAA-compliant applications.
* AWS compliance resources – This collection of workbooks and guides might apply to your industry and location.
* Evaluating Resources with AWS Config Rules - assesses how well your resource configurations comply with internal practices, industry guidelines, and regulations.
#### Acceptance Criteria
Define a compliance strategy.
